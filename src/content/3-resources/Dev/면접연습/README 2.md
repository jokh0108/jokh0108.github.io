# 프론트엔드 개발자 면접 연습 질문 모음

이 폴더에는 프론트엔드 개발자 면접에서 자주 묻는 질문들과 그에 대한 모범 답변, 꼬리 질문 및 참고 자료를 정리했습니다.
각 문서는 메인 질문 하나와 꼬리 질문 세 개를 담고 있으며, 실제 면접에서 유용하게 활용할 수 있는 내용으로 구성되어 있습니다.

## 목차

1. [브라우저 렌더링 과정](01_브라우저_렌더링_과정.md)
   - Critical Rendering Path란?
   - 리플로우(Reflow)와 리페인트(Repaint)의 차이점
   - 브라우저 렌더링 성능 개선 방법

2. [JavaScript 이벤트 루프](02_JavaScript_이벤트_루프.md)
   - 마이크로태스크와 매크로태스크의 차이점
   - 비동기 처리 방식의 발전 과정(콜백, Promise, async/await)
   - JavaScript의 비동기 처리가 필요한 이유

3. [웹 보안 취약점](03_웹_보안_취약점.md)
   - Content Security Policy(CSP)란?
   - Same-Origin Policy와 CORS의 관계
   - JWT(JSON Web Token)의 보안 취약점과 안전한 사용법

4. [웹 성능 최적화](04_웹_성능_최적화.md)
   - Core Web Vitals의 주요 지표와 개선 방법
   - 브라우저 캐싱 전략과 ServiceWorker를 활용한 오프라인 지원 방법
   - 코드 분할(Code Splitting)과 트리 쉐이킹(Tree Shaking)의 차이점과 구현 방법

5. [HTTP 버전 차이점](05_HTTP_버전_차이점.md)
   - HTTP/2의 서버 푸시(Server Push) 기능과 활용 사례
   - HTTPS 동작 원리와 SSL/TLS 핸드셰이크 과정
   - REST API와 GraphQL의 차이점 및 각각의 장단점

6. [JavaScript 메모리 관리](06_JavaScript_메모리_관리.md)
   - 클로저와 메모리 누수의 관계
   - JavaScript에서 흔히 발생하는 메모리 누수의 원인과 해결 방법
   - WeakMap과 Map의 차이점

## 사용 방법

1. 각 주제별 문서를 열어 질문과 답변을 학습합니다.
2. 도식화된 그림을 통해 개념을 시각적으로 이해합니다.
3. 참고 자료 링크를 통해 더 깊이 있는 학습을 진행합니다.
4. 꼬리 질문을 통해 해당 주제에 대한 이해도를 높입니다.

## 기여 방법

질문과 답변을 추가하거나 수정하려면:
1. 새로운 마크다운 파일을 생성하거나 기존 파일을 수정합니다.
2. 문서 형식을 따라 질문, 답변, 참고 자료, 도식화를 추가합니다.
3. README.md 목차에 새로운 문서를 추가합니다.
